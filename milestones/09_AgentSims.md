## 核心架构与技术原理

AgentSims 是一个专为**大语言模型能力评测（Task-based LLM Evaluation）**设计的交互式、可视化多智能体沙盒系统。它旨在解决传统 LLM 评测基准（如 MMLU、GSM8K）容易被训练集污染、评测维度单一且指标不够客观的问题。其架构包含以下核心组件：

```
┌────────────────────────────────────────────────────────┐
│                   AgentSims Frontend                   │
│   ┌────────────────────────────────────────────────┐   │
│   │             Unity-based Interactive GUI        │   │
│   │      (Drag-and-Drop Buildings, Agents, Mayor)  │   │
│   └────────────────────────▲───────────────────────┘   │
└────────────────────────────┼───────────────────────────┘
                             │ (Real-time Socket Sync)
┌────────────────────────────▼───────────────────────────┐
│                   AgentSims Backend                    │
│   ┌────────────────────────────────────────────────┐   │
│   │                 Python Engine                  │   │
│   │   ┌──────────────┐ ┌──────────┐ ┌────────────┐ │   │
│   │   │Memory System │ │ Planning │ │  Tool-Use  │ │   │
│   │   └──────────────┘ └──────────┘ └────────────┘ │   │
│   └────────────────────────▲───────────────────────┘   │
└────────────────────────────┼───────────────────────────┘
                             │ (Abstracted LLMCaller)
┌────────────────────────────▼───────────────────────────┐
│                      Any LLM API                       │
└────────────────────────────────────────────────────────┘
```

### 1. 零代码可视化编辑器（Unity-based GUI）
* 传统的 AI 小镇（如 Stanford Smallville）需要开发者具备较强的编程能力来配置地图和智能体。
* AgentSims 提供了类似《模拟城市》（SimCity）的拖拽式图形界面（基于 Unity 开发）。非计算机专业的社会学、心理学研究者可以通过 GUI 轻松添加建筑物、配置 Agent 的初始人设。

### 2. 模块化认知组件（Pluggable Support Systems）
* 将 Agent 的认知系统抽象为三个可插拔的 Python 类：**记忆系统（Memory）**、**规划系统（Planning）**和**工具调用系统（Tool-Use）**。
* 开发者只需编写几行 Python 代码，就能替换特定的认知算法（例如将向量检索记忆替换为关系型记忆），并在沙盒中直观评测该算法对 Agent 任务完成率的影响。

### 3. 上帝模式与市长机制（Mayor Mode）
* 原生支持人类玩家作为“市长（Mayor）”介入游戏。
* 市长可以动态发布突发事件（如“水源污染”、“商品涨价”），或者直接向 Agent 发送指令，观察 Agent 的应对和决策能力。

## 关键贡献与对社会模拟的启示

* **评测范式的革新**：倡导“基于任务的社会经济模拟评测（Task-based Socio-economic Evaluation）”。通过让 Agent 在虚拟小镇中生存、交易、社交，来综合评估 LLM 的长期规划、资源管理和人际沟通能力。
* **跨学科民主化**：通过 Unity 前端与 Python 后端的紧密结合，降低了社会科学研究者利用大模型进行“计算社会学”实验的门槛。
