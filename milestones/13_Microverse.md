# 13. Microverse: 基于Godot 4的上帝视角AI社会沙盒

## 基础信息
* **时间**：2025年10月
* **来源**：
  * **开源仓库**：[KsanaDock/Microverse](https://github.com/KsanaDock/Microverse)
  * **开发引擎**：Godot 4.3+ (GDScript)

---

## 核心架构与技术原理

Microverse（微观宇宙）是一个基于 **Godot 4 引擎** 开发的开源、上帝视角（God Game）多智能体 AI 社会模拟沙盒系统。它在工程实现上，将现代游戏引擎的高性能渲染、物理碰撞与大语言模型的认知决策进行了深度融合。

### 1. 基于 Godot 4 的高性能底座
* 相比于基于 Web（如 PixiJS）的 AI Town，Microverse 采用专业的游戏引擎 Godot 4 开发。
* GDScript 编写的底层逻辑能够更高效地处理 Agent 的寻路（Navigation2D/3D）、碰撞体积、视线感知（RayCast）以及复杂的场景物品交互，支持在单机上流畅运行更多的并发 Agent。

### 2. 多模型服务原生集成（Multi-Provider Integration）
* 原生支持接入全球主流的大模型 API，包括 OpenAI、Claude、Gemini、DeepSeek 以及国内的豆包、Kimi 等。
* 开发者和玩家可以在游戏内随时切换驱动 Agent 的模型，直观对比不同模型在社交、任务执行和记忆维持上的表现差异。

### 3. 自主任务管理系统（Autonomous Task Management）
* 引入了动态的任务发布、分配、执行与竞争机制。
* Agent 能够根据自身的人格、当前需求和环境状态，自主决定是否接受某个任务（如“去农田收割”、“去酒馆打听消息”），并能与其他 Agent 协作或竞争完成任务。

### 4. 连续持久化记忆与对话回放
* 采用本地 JSON 格式进行轻量级、持久化的记忆存储。
* 提供了完整的对话历史记录和回放功能，玩家可以像看录像一样回溯任意两个 Agent 在过去某天某时的详细对话过程。

---

## 关键贡献与对社会模拟的启示

* **游戏引擎与 AI 的深度融合**：展示了如何利用现代开源游戏引擎（Godot）的生态，快速构建高性能、可交互的 AI 虚拟社会。
* **低成本、多模型的评测沙盒**：由于其广泛的模型支持，Microverse 成为了评估不同大模型在“具身社交与任务执行”场景下实际表现的绝佳物理沙盒，推动了开源 AI 社区在多智能体方向的探索。
