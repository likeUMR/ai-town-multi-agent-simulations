AI 小镇与多智能体社会模拟
三大分类与技术演进全景报告

大模型多智能体（LLM Multi-Agent）社会模拟自2023年爆发以来，已经从学术界玩具演进为跨学科的科学工具和商业原生游戏。本报告将 19 个里程碑节点结构化地分类为**为了模拟**、**为了开发**和**为了游戏**三大板块，帮助您在组会汇报中层层递进、逻辑闭环。

为了模拟 (Simulation) 📊

计算社会学与政策评估

核心定位：把 Agent 当作人类社会的显微镜。通过模拟宏观群体行为的演化，来预测、解释、干预真实人类社会的规律。

● 技术侧重：记忆-反思-规划认知架构、分布式调度
● 评估标准：实证对齐（统计学一致性）
● 代表作：生命游戏, Stanford小镇, AI Town, AgentSociety
为了开发 (Development) ⚙️

生产力工具与工作流自动化

核心定位：把 Agent 当作专业打工人。不关心 Agent 的日常生活，只关心如何高效、低成本、高确定性地解决复杂任务。

● 技术侧重：强约束协作、SOP（标准作业程序）、工具调用
● 评估标准：任务成功率、Token 消耗成本
● 代表作：CAMEL, ChatDev, MetaGPT, WorldCraft
为了游戏 (Gaming) 🎮

智能NPC与动态剧情自涌现

核心定位：把 Agent 当作活着的虚拟伴侣与造物主。关注玩家的沉浸感与可玩性，NPC 拥有私心、立场与真实行为。

● 技术侧重：语义与数值/物理打通、编剧Agent、低延迟
● 评估标准：可信度、可玩性、玩家沉浸感
● 代表作：WorldBox, AIvilization, MagiScapes, Astrobuilder

🔺 智能体三元悖论 (The Trilemma of Agents)

在多智能体系统设计中，存在一个类似区块链的**“三元悖论”**：任何一个系统都很难同时完美兼顾以下三个维度，必须根据其分类定位进行取舍。

高确定性 (Determinism)

要求行为严格可控，无幻觉，按预设逻辑输出。“为了开发”的系统极度追求此项，通过强 SOP 限制 Agent 自由度。

高拟真多样性 (Alignment & Variety)

要求 Agent 展现出丰富的人类性格、自发社交和宏观涌现。“为了模拟”的系统极度追求此项，允许犯错和随机性，但绝不能性格同质化。

极低延迟与成本 (Low Latency & Cost)

要求高并发、秒级响应和低运行成本。“为了游戏”的商业化产品极度追求此项，往往需要用轻量端侧模型（如 Qwen-7B）做极端优化。

⚖️ 三大维度核心技术对比矩阵

对比维度 为了模拟 (Simulation) 为了开发 (Development) 为了游戏 (Gaming)
核心追求 宏观社会规律涌现、科学预测 任务成功率、生产力提效 玩家沉浸感、NPC可信度、可玩性
Agent 自由度 高（允许犯错、自由社交） 极低（受 SOP 与流程强约束） 中等（在游戏规则边界内自由决策）
核心技术栈 记忆-反思-规划、Ray分布式、向量检索 工具调用、代码执行、瀑布流研讨会 语义与数值打通、编剧Agent、端侧模型
算力与延迟 高延迟、极高成本（离线异步运行） 中延迟、中成本（按需单次调用） 极低延迟、低成本（实时并发运行）
评估与对齐 实证对齐（大五人格、统计学检验） 代码单元测试、Benchmark 跑分 玩家主观体验、留存率、UGC生成质量
